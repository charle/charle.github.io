添加路由

添加路由的方法

  • sudo route add -net 10.28.131.0 netmask 255.255.255.0 gw 10.61.2.2
    • 10.28.131.0 #表示destiny路由网段
    • 255.255.255.0 #表示掩码
    • gw #网关
    • 10.61.2.2 #网关地址

删除路由的方法

  • sudo route del -net 10.28.131.0 netmask 255.255.255.0 gw 10.61.2.2
    • 用法类似添加路由

vim的使用

批量替换

： %s /10.61.13.25/eth0/g #全局替换
：s/10.61.13.25/eth0/g #当前行替换
： 1,15 s/10.61.13.25/eth0/g #1-15替换
visual模式选择你要替换的部分 s/10.61.13.25/eth0/g

vim其它常用操作

  • 删除命令
    • d0 #删至行首
    • d$ #删至行尾
    • ndd #删除当前行及后n-1行
    • x或X #删除一个字符，x删除光标后的，而X删除光标前的
    • dd #删除当前行
  • 移动光标命令
    • ） #光标移至句尾
    • （ #光标移至名首
    • } #光标移至段落开头
    • { #光标移到段落结尾
    • nG # 光标移至第n行首
    • n+ #光标下移n行
    • n- #光标上移n行
    • n$ #光标移至第n行尾
    • H #光标移到屏幕顶行
    • M #光标移至屏幕中间行
    • L #光标移到屏幕最后行
    • 0 #光标移至当前行首
    • $ #光标移至当前行尾

session,cookie的区别

存储位置不同

  • session是存储在服务器的
  • cookie是存储在客户端的

原理不同

  • cookie机制采用的是在客户端保持状态的方案，它是在用户端的会话状态的存贮机制，他需要用户打开客户端的cookie支持，cookie的作用就是为了解决http协议无状态的缺陷所作的努力。
  • session是针对每一个用户的，变量的值保存在服务器上，用一个sessionID来区分是哪个用户session变量，这个值是通过用户的浏览器在访问的时候返回给服务器，当客户禁用cookie时，这个值也可能设置为由get来返回给服务器

git使用

提纲

  • 查看远程分支
  • 删除远程分支和tag
  • 删除不存在对应远程分支的本地分支
  • 重命名远程分支
  • 把本地tag推送到远程
  • 获取远程tag

查看远程分支

加上-a参数可以查看远程分支，远程分支会用红色表示出来

git branch -a

删除远程分支和tag

在git v1.7.0之后，可以使用这种语法删除远程分支

git push origin —delete branchName

删除tag可以这么用

git push origin —delete tag tagName

把本地tag推送到远程

git push —tags

获取远程tag

git fetch origin tag tagName
