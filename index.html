tail-wc-head使用

wc使用

  • wc -c filename #显示一个文件的字节数
  • wc -m filename #显示一个文件的字符数
  • wc -l filename #显示一个文件的行数
  • wc -L filename #显示一个文件中最长行的长度
  • wc -w filename #显示一个文件了字数
  • wc -lcw file1 file2 #组合使用

head使用

  • head -n 5 filename #显示文件前5行
  • head -c 20 filename #显示文件的前n个字节
  • head -c -32 filename #显示文件除了最后n个字节以外的内容
  • head -n -6 filename #输出文件除了最后n行的全部内容

tail使用

  • tail -n 5 filename #显示文件末尾5行内容
  • tail -f filename #循环查看文件内容
  • tail -n +5 filename #从第5行开始显示文件

mysql乱码

打开终端登陆到服务器，登陆mysql数据库,查询数据总是乱码，一开始觉得是终端编码设置有问题，没放在心上，后来网上搜了些东西，原来mysql也可以设置编码格式输出的

mysql输出乱码命令：

mysql> set names ‘gbk’;如果不行则可能用 set names ‘utf8’;

php输出乱码命令

mysql_query(“set name ‘gbk’”);

phpmyadmin里乱码

phpMyAdmin的config.inc.php中有没有设置$cfg[‘DefaultCharset’]=’utf-8’;

添加路由

添加路由的方法

  • sudo route add -net 10.28.131.0 netmask 255.255.255.0 gw 10.61.2.2
    • 10.28.131.0 #表示destiny路由网段
    • 255.255.255.0 #表示掩码
    • gw #网关
    • 10.61.2.2 #网关地址

删除路由的方法

  • sudo route del -net 10.28.131.0 netmask 255.255.255.0 gw 10.61.2.2
    • 用法类似添加路由

vim的使用

批量替换

： %s /10.61.13.25/eth0/g #全局替换
：s/10.61.13.25/eth0/g #当前行替换
： 1,15 s/10.61.13.25/eth0/g #1-15替换
visual模式选择你要替换的部分 s/10.61.13.25/eth0/g

vim其它常用操作

  • 删除命令

    • d0 #删至行首
    • d$ #删至行尾
    • ndd #删除当前行及后n-1行
    • x或X #删除一个字符，x删除光标后的，而X删除光标前的
    • dd #删除当前行

  • 移动光标命令

    • ） #光标移至句尾
    • （ #光标移至名首
    • } #光标移至段落开头
    • { #光标移到段落结尾
    • nG # 光标移至第n行首
    • n+ #光标下移n行
    • n- #光标上移n行
    • n$ #光标移至第n行尾
    • H #光标移到屏幕顶行
    • M #光标移至屏幕中间行
    • L #光标移到屏幕最后行
    • 0 #光标移至当前行首
    • $ #光标移至当前行尾

session,cookie的区别

存储位置不同

  • session是存储在服务器的
  • cookie是存储在客户端的

原理不同

  • cookie机制采用的是在客户端保持状态的方案，它是在用户端的会话状态的存贮机制，他需要用户打开客户端的cookie支持，cookie的作用就是为了解决http协议无状态的缺陷所作的努力。
  • session是针对每一个用户的，变量的值保存在服务器上，用一个sessionID来区分是哪个用户session变量，这个值是通过用户的浏览器在访问的时候返回给服务器，当客户禁用cookie时，这个值也可能设置为由get来返回给服务器
